Preparing Network Traffic for Wireshark using TCPDump.

This is a step by step walkthrough for the live capture.

1. The first step is to create a shell for the live capture and decide how many packets we will capture for Wireshark. We will use analyze 25 packets from a sample website called skyroute66 using TCPDump. When writing the shell script we will use sudo for special permission, and lower case -c followed by 25 for the count of packets being captured. The [-##XXtttt] command will allow our capture to be organized by column for a better analysis experience. Lastly, we will use the write command -w followed by the file name we want to call it. We can call the pcap file wscapture. The shell script will look like:
sudo tcpdump -XXtttt host skyroute66.com -c 25 -w wscapture.cpap

3. By using the write command and wscapture the file is created and we can now open and better analyze the traffic in Wireshark. Symbols, protocol, and destination information can be studied in Wireshark to triage tickets and/or for research.
